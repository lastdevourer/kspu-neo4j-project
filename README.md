# KSU Academic Network

Інформаційно-аналітична система для обліку, перегляду та аналізу наукових публікацій викладачів KSU на базі `Streamlit` і `Neo4j Aura`.

## Призначення системи

Система створена для:

- зберігання відомостей про факультети, кафедри, викладачів і публікації;
- аналізу авторства та співавторства;
- побудови графових зв'язків між викладачами, публікаціями та підрозділами;
- формування аналітичних зрізів і звітів;
- підтримки ручної перевірки та уточнення даних.

## Джерела даних

Основним джерелом вихідних даних є відкрито оприлюднені сторінки офіційного сайту ХДУ з відомостями про публікації у `Scopus` та `Web of Science`.

- [Публікації науковців ХДУ](https://www.kspu.edu/About/DepartmentAndServices/Library/Actual/Academics/PublicationsKSUscientistsNMBD.aspx)
- [Scopus: публікації науковців ХДУ](https://www.kspu.edu/About/DepartmentAndServices/Library/Actual/Academics/PublicationsKSUscientistsNMBD/ScopusKSU.aspx)
- [Web of Science: публікації науковців ХДУ](https://www.kspu.edu/About/DepartmentAndServices/Library/Actual/Academics/PublicationsKSUscientistsNMBD/WebOfScienceKSU.aspx)

Підтримка зовнішніх сервісів (`ORCID`, `OpenAlex`, `Crossref`, `Scopus`, `Web of Science`, `Google Scholar`) підготовлена в архітектурі системи та може використовуватися як додатковий спосіб пошуку й уточнення даних після налаштування доступу.

## Що реалізовано

- дашборд із загальними показниками структури та публікацій;
- каталог викладачів і каталог публікацій;
- граф авторства, співавторства та міжкафедральних зв'язків;
- аналітика, рейтинги, порівняльні зрізи та звіти;
- модуль керування структурою університету;
- центр даних для ручної перевірки, журналу запусків і роботи з проблемними записами;
- базове завантаження публікацій з відкритих сторінок ХДУ;
- підготовлені інтеграції із зовнішніми сервісами для додаткового пошуку даних.

## Режими завантаження публікацій

У системі передбачено два окремі сценарії роботи з публікаційними даними.

### 1. Завантаження з відкритих сторінок ХДУ

Використовується як основний сценарій демонстрації:

- читає відкриті сторінки ХДУ;
- формує записи публікацій;
- створює зв'язки авторства у графовій базі.

### 2. Додатковий пошук даних

Може використовуватися після налаштування доступу до зовнішніх сервісів:

- `ORCID`
- `OpenAlex`
- `Crossref`
- `Scopus`
- `Web of Science`
- `Google Scholar`

Цей режим не є обов'язковим для базової роботи системи, але підготовлений як напрям подальшого розширення.

## Режими доступу

### Публічний режим

Доступні сторінки:

- `Дашборд`
- `Граф`
- `Аналітика`
- `Викладачі`
- `Публікації`

У публічному режимі приховано службові сторінки та дії, пов'язані з редагуванням і модерацією.

### Режим керування

Додатково відкриваються:

- `Структура`
- `Центр даних`

Для входу використовується пароль у `Streamlit Secrets`:

```toml
ADMIN_PASSWORD = "your-password"
```

Параметр `ADMIN_MODE=true` можна використовувати лише для службового запуску, якщо потрібно тимчасово відкрити режим керування без введення пароля.

## Запуск локально

```bash
streamlit run app.py
```

## Secrets для Streamlit Cloud

```toml
NEO4J_URI = "neo4j+s://..."
NEO4J_USER = "neo4j"
NEO4J_PASSWORD = "..."

# необов'язкові параметри для додаткового пошуку даних
OPENALEX_API_KEY = "..."
CROSSREF_MAILTO = "you@example.com"
ORCID_CLIENT_ID = "..."
ORCID_CLIENT_SECRET = "..."
SCOPUS_API_KEY = "..."
SCOPUS_INSTTOKEN = "..."
WOS_API_KEY = "..."
PUB_IMPORT_MAX_WORKS_PER_TEACHER = 40

# режим керування
ADMIN_PASSWORD = "your-password"
ADMIN_MODE = false
```

Для базового сценарію достатньо параметрів `NEO4J_*` та `ADMIN_PASSWORD`.

## Структура проєкту

- `app.py` — точка входу.
- `full_app.py` — маршрутизація сторінок.
- `config.py` — конфігурація, secrets, параметри підключення.
- `views/` — сторінки інтерфейсу.
- `ui/` — стилі, компоненти, sidebar, форматування таблиць.
- `services/neo4j_service.py` — робота з графовою базою.
- `services/publication_import.py` — логіка завантаження та опрацювання публікацій.
- `services/publication_sources.py` — робота із зовнішніми джерелами.
- `utils/` — аналітика та графова візуалізація.
- `data/` — початкові дані та шаблони.
- `docs/data_pipeline.md` — короткий опис логіки наповнення.
- `scripts/` — допоміжні скрипти.

## Основні сторінки

- `Дашборд` — загальний стан системи.
- `Викладачі` — пошук і перегляд карток викладачів.
- `Публікації` — каталог наукових робіт.
- `Граф` — мережеві зв'язки між авторами, публікаціями та підрозділами.
- `Аналітика` — звіти, рейтинги, динаміка, порівняльні зрізи.
- `Структура` — керування факультетами, кафедрами, викладачами та публікаціями.
- `Центр даних` — ручна перевірка, журнал запусків, робота з проблемними записами.
