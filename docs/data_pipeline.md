# Пайплайн наповнення даних

## Пріоритет джерел

1. Офіційний сайт ХДУ (`kspu.edu`)
   Використовується для структури `Faculty -> Department -> Teacher` та як базове джерело відкритих даних про публікації науковців.
2. Відкриті сторінки бібліотеки ХДУ з публікаціями у `Scopus` та `Web of Science`
   Саме вони розглядаються як основний вихідний масив для демонстраційного та дипломного сценарію.
3. ORCID / Google Scholar / Scopus / Web of Science профілі
   Використовуються як профільні посилання викладачів і як опора для валідації.
4. OpenAlex / Crossref / ORCID API
   Розглядаються як додаткове автоматичне збагачення, а не як обов'язкова основа системи.

## Чому не варто робити ставку лише на Google Scholar

- Google Scholar погано підходить для стабільного автоматичного парсингу.
- Профілі можуть бути неофіційними, неповними або дубльованими.
- Краще зберігати `google_scholar` як посилання для валідації, а зовнішні сервіси використовувати лише для збагачення вже зібраного відкритого контуру.

## Рекомендований порядок наповнення

1. Завантажити факультети та кафедри з офіційного сайту.
2. Зібрати список викладачів по кафедрах.
3. Зафіксувати відкриті сторінки ХДУ з публікаціями у `Scopus` та `Web of Science` як основне джерело вихідних даних.
4. За потреби прив'язати викладачів до опублікованих на сайті записів вручну або напівавтоматично.
5. Лише після цього запускати зовнішнє збагачення через профільні сервіси.
6. Для кожного викладача зафіксувати:
   - повне ім'я
   - кафедру
   - посаду
   - науковий ступінь
   - ORCID
   - Google Scholar
   - Scopus
   - Web of Science
   - посилання на сторінку кафедри або профіль
7. Завантажувати публікації та авторство окремо, щоб легше контролювати якість даних.

## Формулювання для дипломного сценарію

Джерелом вихідних даних у роботі є відкрито оприлюднені сторінки офіційного сайту ХДУ з відомостями про публікації у `Scopus` та `Web of Science`. API-інтеграції залишаються додатковим засобом автоматичного збагачення та можливим напрямом подальшого розвитку системи.

## Мінімальна схема імпорту

- `teachers.csv`
- `publications.csv`
- `authorships.csv`

## Практичний наступний етап

Для MVP варто почати з 2-3 кафедр і 20-30 викладачів:

- кафедра комп'ютерних наук та програмної інженерії
- кафедра хімії та фармації
- кафедра англійської філології та світової літератури імені професора Олега Мішукова

Цього вже вистачить, щоб граф, пошук, co-authorship та centrality виглядали переконливо на захисті.
