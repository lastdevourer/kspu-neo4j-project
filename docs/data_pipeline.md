# Пайплайн наповнення даних

## Основна ідея

Система працює у два етапи:

1. базове завантаження даних з відкритих сторінок офіційного сайту ХДУ;
2. за потреби — додатковий пошук даних через зовнішні сервіси.

Такий підхід дозволяє використовувати систему навіть без зовнішніх API-ключів.

## Пріоритет джерел

1. Офіційний сайт ХДУ  
   Використовується для побудови структури `Faculty -> Department -> Teacher`.

2. Відкриті сторінки ХДУ з публікаціями у `Scopus` та `Web of Science`  
   Це основний масив вихідних даних для демонстраційного та дипломного сценарію.

3. Профільні посилання викладачів  
   `ORCID`, `Google Scholar`, `Scopus`, `Web of Science` можуть використовуватися для уточнення профілів.

4. Зовнішні API  
   `OpenAlex`, `Crossref`, `ORCID API` та інші джерела підключаються лише як додатковий спосіб пошуку даних.

## Рекомендований порядок роботи

1. Завантажити факультети, кафедри та викладачів.
2. Запустити базове завантаження публікацій з відкритих сторінок ХДУ.
3. Перевірити результати в `Публікації`, `Аналітика` та `Граф`.
4. За потреби виконати ручне уточнення або повторне зіставлення записів.
5. Лише після цього запускати додатковий пошук даних через зовнішні сервіси.

## Формулювання для дипломної роботи

Джерелом вихідних даних у роботі є відкрито оприлюднені сторінки офіційного сайту ХДУ з відомостями про публікації у `Scopus` та `Web of Science`. Інтеграції із зовнішніми сервісами реалізовані в архітектурі системи та можуть використовуватися як додатковий напрям розвитку після налаштування доступу.

## Мінімальний набір даних

- `teachers.csv`
- `publications.csv`
- `authorships.csv`

## Практичний результат

У базовому режимі система дозволяє:

- зібрати дані про публікації;
- зіставити їх із викладачами;
- побудувати зв'язки авторства;
- сформувати графи, каталоги та аналітичні звіти.
